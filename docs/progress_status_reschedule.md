# progress / status / reschedule メモ

## `/progress`

- 作品名 / 工程名 / 実績時間 / 状態 / イベント名 / メモ で実績を SQLite に保存します
- `/estimate` で保存した工程見積があれば、見積との差分も返します

## `/status`

- 直近の見積スナップショットと実績をもとに、危険度が高い作品を確認できます
- 危険度は、締切までの日数・未完了工程数・実績超過をもとに簡易判定します

## `/reschedule`

- 締切・未完了工程・実績差分を見て、優先着手候補や後ろ倒し候補を返します
- Google カレンダー連携が設定されていれば、予定圧も補足として表示します

## 補足

- 実績反映はまず SQLite 中心です
- Notion の進捗自動更新はまだ行いません
- `/event` `/task` `/estimate` の既存導線はそのまま残しています
